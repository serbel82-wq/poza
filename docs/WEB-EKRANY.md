# Веб-клиент: экраны и потоки (MVP)

## Роли

- **Ребёнок** — основной интерфейс обучения.
- **Родитель** — оплата, прогресс, настройки аккаунта.

Один логин может вести к выбору профиля, если позже будет несколько детей; в MVP достаточно **один ребёнок на аккаунт** для упрощения.

---

## Потоки

### A. Регистрация и первый вход

1. **Landing** — что это, для кого, CTA «Начать», цена/триал кратко.
2. **Регистрация** — email родителя, пароль, имя ребёнка, возраст (10–14), согласие с документами.
3. **Подтверждение email** (опционально в MVP можно отложить — с оговоркой по безопасности; лучше включить до платежей).
4. **Онбординг** — приветствие персонажа/бренда, 3–5 карточек «правила безопасности», кнопка «Понял».
5. **Домашняя (ребёнок)** — карточка «Сезон 1», кнопка «Продолжить» / список уроков.

### B. Урок

1. **Список уроков сезона** — номер, название, статус (закрыт/открыт/готово).
2. **Экран урока** — блоки: цели → теория → практика → «Отправить» / «Дальше».
3. **Экран завершения урока** — бейдж/галочка, краткое «что дальше».

### C. Проект сезона

1. **Вводный экран проекта** — выбор трека A/B/C.
2. **Пошаговый мастер** — поля по шаблону, сохранение черновика.
3. **Сдача** — кнопка «Завершить сезон», поздравление.

### D. Родитель

1. **Вход** — тот же аккаунт или отдельная вкладка «Я родитель» (можно один логин).
2. **Сводка** — имя ребёнка, % сезона, последняя активность, тема текущего урока.
3. **Подписка** — статус, дата списания, отмена, история платежей (минимум).
4. **Документы** — ссылки на политику и оферту.

### E. Платёж

1. **Оформление подписки** — после триала или сразу (по твоей модели).
2. **Успех / ошибка** — редирект на родительскую сводку.

### F. Служебное

1. **404 / ошибка**
2. **Техработы** (заглушка)
3. **Выход из аккаунта**

---

## Приоритет отрисовки (если дизайн один на всё)

1. Домашняя ребёнка + экран урока.
2. Регистрация + landing.
3. Родительская сводка + подписка.

---

## Заметки для мобилки «потом»

- Те же маршруты; усилить touch-цели и офлайн-чтение теории (PWA).
